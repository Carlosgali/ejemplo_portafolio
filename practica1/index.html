Arduino Electrónica

- Resumen -

En la siguiente practica se realizo una comparacion entre tres dispositivos diferentes, Atiny45, Arduino, y SP32, realizando pruebas, las cuales consistian en utilizar el mismo codigo para encender un led, hacerlo parpadear, y mediante un boton encender el led, y asi identificar las capacidades de cada uno.

A continuacion se vere una explicacion mas extensa

- Introducción -

Estos circuitos integrados compactos (IC) contienen un núcleo de procesador (o núcleos), memoria de acceso aleatorio (RAM) y memoria de solo lectura programable borrable eléctricamente (EEPROM) para almacenar los programas personalizados que se ejecutan en el microcontrolador, incluso cuando la unidad está desconectada de una fuente de alimentación.

A diferencia de los microprocesadores de uso general, los microcontroladores integran periféricos de procesamiento, memoria y entrada/salida (E/S), incluidos temporizadores, contadores y convertidores de analógico a digital (ADC), en una unidad independiente eficiente y rentable. Al combinar múltiples componentes en un solo sistema, los microcontroladores son adecuados para aplicaciones que requieren procesamiento de señales en tiempo real, como controlar motores y servos e interactuar con varios tipos de sensores y comunicaciones.

- Desarrollo -

Se realizo la programación en el simulador Wokwi, también se realizó la simulación virtual conectando cada elemento de acuerdo con sus respectivas especificaciones, una vez que se visualizo su funcionamiento, se replicó con cada uno de los microdispositivos, una vez hechas las pruebas y al ver que todo esta en orden, se realizaron las pruebas físicas y se tomaron las respectivas evidencias.

.

Simulación

La simulación se llevó a cabo con el programa Wokwi, para realizar el código y poderlo visualizar mediante el ensamble virtual de los componentes que después se montarían de manera física.

.

Diseño

Para el diseño de las practicas, se montaron individualmente para no generar errores en las conexiones, así que se opto por tomas las primeras filas para conectar el microcontrolador, y así tener fotos y videos exclusivas por cada experimento realizado.

.

Construcción

Primero se opto por elegir que componente se utilizaría para así evaluar cada uno de los elementos a utilizar, una vez que ya se tenían identificados cada uno de estos se procedió a conectarlos sobre el protoboard, antes de conectar el microcontrolador. Una vez verificado que todo estuviera en orden, se procedió a conectar con la interfaz para poder cargar el código y verificar su funcionamiento, una vez acreditado el buen funcionamiento, se procedió a realizar cada ensamble conforme a los requerimientos, comenzando con el led, encendido, el parpadeo del led, el push boton con su respectivo while.

.

- Resultados -

Aqui encontraremos una serie de videos, que nos permitiran visualizar los 4 experimentos realizados, para poderlos comparar de forma visual.

.

- Conclusiones -

Pudimos concluir que cada microcomponente funciona de formas distintas, realizando las tareas para el cual fue desarrollado, Arduino al ser un microcomponente que se utiliza regularmente para muchos proyectos, se encuentra fácilmente a un precio accesible y se conoce mucho sobre el habiendo las practicas un poco mas sencillas, Atiny, al ser un dispositivo muy pequeño está limitada su capacidad de procesamiento, y realiza las mismas tareas que se diseñaron de una manera mas torpe, ESP32 es el dispositivo mas potente de los tres, realizando las mismas tareas de una manera mas fluida, aunque los ejercicios que se realizaron no fueron muy demandantes, se pusieron notar pequeñas diferencias en la velocidad procesamiento entre los tres.
